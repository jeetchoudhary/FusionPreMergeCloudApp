Pre-merge is a process which executes several script written specifically to checks our code for common Code_smell , which are not desirable application code base. Apart from the basic functinality it also compiles the code and builds the project to ensure that we are not introducing compile time failures along with some other activities which are part of continous integration e.g Automated Unit Test , ant build etc. As of now the pre-merge tool runs on individual developer machines and update the bug with the final results. This will have an overhead on the client machines memory and makes the machine performance slow. This Tool abstract the use of Pre-merge from the developer .This will accept transaction name from the user , then it will perform per-merge scripts on the transaction , update the bug in the bug DB and send mail to the requestor. This whole process will be done on dedicated configured VM's so that there is no overhead on developer machine.
For more information on this to this tool please refer to the documentation.
